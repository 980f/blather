LEGO Old IR protocol

Modulation is 76kHz, twice the present day 38kHz.
Baud is 4800.
Two bytes are sent, each is 8O1.

There is a talk of "25% duty cycle" which I gather means "don't send back to back messages, wait 10X3 bit times between messages". One suspects that was to allow the receivers to be very slow at dealing with messages.

First byte MS nibble is channel, 4 for all, else 4+[1,2,3].
First byte LS nibble is orange control (right hand buttons) is 0:no press,7:forward,8:"off", or F:reverse. I am guessing "off" is sent by pressing both buttons together.
Second byte MS nibble is yellow control (left hand buttons).

Second byte LS nibble is a check digit: 0 - (address + yellow + orange).
Checking should yield 16 when you add the 3 other nibbles.

Message rates are somewhat slow:
Ch. 1 - 19.6 Hz
Ch. 2 - 14.8 Hz
Ch. 3 - 11.6 Hz
‘All’ - 19.6 Hz
This appears to be to reduce collisions when multiple independent transmitters are active, a single transmitter controlling 6 motors can send at the 19.6 for all.
